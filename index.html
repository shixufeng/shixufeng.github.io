编写程序，从标准输入读入一系列string对象，寻找连续重复次数最多的单词，并输出查找信息。 

// 从标准输入读入一系列string对象，寻找连续重复出现的单词
// 输出重复次数的最大值及对应的单词
// Language : C++

#include <iostream>
#include <string>

using namespace std;

int main ()
{
    // 计数器---当前字符串计数器curCnt，重复最多次字符串计数器maxCnt
    int curCnt = 1, maxCnt = 1;

    // 字符串---前一字符串preStr，当前字符串curStr，重复最多次字符串maxNumStr
    string preStr, curStr, maxNumStr;
    cout << "Enter first string (Ctrl+Z to end) : ";
    cin >> preStr;
    cout << endl;
    maxNumStr = preStr;

    cout << "Enter your current string (Ctrl+Z to end) : " << endl;
    // 判断连续重复字符串并计数
    while (cin >> curStr)
    {
        // 与前一字符串重复，计数器+1，并判断重复次数最多的字符串
        if (curCnt == preStr)
        {
            ++ curCnt;
            if (curCnt > maxCnt)    // 重复次数最多，更新maxCnt与maxNumStr
            {
                maxCnt = curCnt;
                maxNumStr = curStr;
            }            
        }
        else    // 与前一字符串不重复，更新preStr，重置计数器curCnt
        {
            preStr = curStr;
            curCnt = 1;
        }
    }

    // 输出连续重复次数最大值与对应字符串
    if (maxCnt >= 2)
    {
        cout << "The information of maxNumStr : " << '\n'
             << maxCnt << "times" << '\t' << maxNumStr << endl;
    }
    else
    {
        cout << "There is no repeated string!" << endl;
    }

    return 0;
}

使用C++编程读取*.txt内容与将程序输出结果输出到*.txt

在学习和工作中，我们常常需要编写程序分析某个文本的特征。如果在程序执行过程中将文本逐个字符输入，效率之低不言而喻，我们自然会想到让程序直接去读取文本内容即可，如何做到这一点呢？下面我们以.txt格式文本为例说明。

C++提供的IO标准库头文件fstream分别定义了从文件中读取与写到文件中去行为的标准类ifstream与ofstream。其中ifstream由istream派生而来，ofstream由ostream派生而来。调用头文件fstream

#include <fstream>

假设被读取文本为file1.txt，写入文本为file2.exe。定义ifstream类型对象fin与ofstream类型对象fout

ifstream fin("file1.txt");     // read from file1.txt
ofstream fout("file2.txt");    // write into file2.txt

由于不知道文本内容类型，定义模板类

template <typename T>
{
    //statements
    ...
}

定义T类型对象tObj，rObj。将文本file1.txt内容写入tObj，将文本分析结果写入rObj进而导入file2.txt

T tObj, rObj;

fin >> tObj;
// statements
...
// results write into rObj
...
// results write into file2.txt
fout << rObj;

通过上述做法便可以实现我们的目标。

调用GNU编译器与微软VS编译器

调用C++编译器的命令因编译器和操作系统的不同而不同，常用的编译器是GNU编译器和微软Visual Studio编译器，调用GNU编译器的命令是g++：

$ g++ prog1.cpp -o prog1

这里的$是系统提示符。这个命令产生一个名为prog1或prog1.exe的可执行文件。在UNIX系统下，可执行文件没有后缀；而在Windows下，后缀为.exe，-o prog1是编译器参数以及用来存放可执行文件的文件名。如果省略-o prog1，那么编译器在UNIX系统下产生名为a.out而在Windows下产生名为a.exe的可执行文件。微软VS编译器采用命令cl来调用：

C:\directory> cl -GX prog1.cpp

这里的C:\directory>是系统提示符，directory是当前目录名。cl是调用编译器的命令，-GX是使用命令行界面编译程序时所需的一个选项。微软VS编译器自动产生与源文件同名且后缀为.exe的可执行文件。

Summary of "Increasing Situation Awareness by Visualizing Uncertainty"

This paper proposes a method for managing 2D visualization to increase situation awareness (SA).

  • Four sources of “information fog”
    • Inadequate or poor quality information
    • Misinterpretation of information
    • Conflicting information or choices
    • Too much information
  • Five key techniques were considered
    • Tailoring the display
    • De-cluttering
    • Integration
    • Cross tool control
    • Use of toolkit
  • Categorize each piece of the battle field into three types
    • Areas where we have no presence and hence no data collection capability
    • Areas where we have collection coverage and no threat data is reported
    • Areas where we have coverage and have collected threat data

The areas of coverage on a overlay may be drawn as semi-transparent, or with a variety of patterned fills such as crosshatches or stripes.

  • Two types of data organization
    • Time
    • Space
  • Proposal for enhancing the display
    • One source type per overlay, plus a universal overlay switch
    • Display sensor coverage for a period of time specified in a user modifiable configuration file
    • Allow the user to set the period of time on a slider, from the maximum specified to immediate
    • Allow coverage segments to be created based on altitude intervals
    • Terrain projection
    • Allow the operator the capability to apply a time stamp and confidence value to any displayed icon
  • Future work
    • Saving individual preferences for visualization techniques
    • Implementing a variety of techniques to ditinguish confidence levels for individual data points
    • Allowing filtering by confidence levels
    • Implementing the inverse coverage display
    • Adding some of the proposed visualization interaction techniques

数据可视化的基本概念

数据可视化的定义
在计算机科学中，数据可视化可定义为对数据进行交互的可是表达以增强认知的技术。

数据可视化的分类
按数据类型分为科学可视化和信息可视化；按可视化目的分为探索性可视化和解释性可视化，前者存在于数据分析阶段，后者存在于视觉呈现阶段。

数据可视化的目标
呈现数据重要特征，揭示客观规律，辅助理解事物。

数据可视化的功能
信息记录、信息推理和分析和信息传播与协同。

数据可视化的要求
真实性、易感知和艺术性。

现代数据的特征
体积大，维度高，动态，多源，多态，趋于非结构化，价值密度低等。
